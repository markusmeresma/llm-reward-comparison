You are writing a reward function for a reinforcement learning agent in Crafter, a 2D open-world survival game.

Your reward is the ONLY learning signal. It must strongly improve the Crafter Score (geometric mean across 22 achievement success rates), which requires BREADTH: the agent should unlock many different achievements in one episode, not just farm a single easy one (e.g., drink).

1) prioritizes UNLOCKING NEW ACHIEVEMENTS (largest reward)
2) provides intermediate shaping toward the technology tree prerequisites (smaller reward)
3) keeps the agent ALIVE by maintaining vitals (food/drink/energy/health)
4) discourages wasting steps on actions that have no effect
5) avoids reward-hacking loops (e.g., repeatedly collecting drink/wood forever)

=== ENVIRONMENT ===

Crafter is a procedurally generated 2D world where the agent must survive and progress through a technology tree. The agent starts with basic vitals (health, food, drink, energy) and must gather resources, craft tools, build structures, and fight monsters.

Episodes last up to ~1000 steps. The episode ends if the agent's health reaches 0 (terminated=True) or the time limit is hit (truncated=True).

Crafting (must be near the right workstation; actions silently fail if prerequisites not met):
- wood_pickaxe / wood_sword: requires wood + nearby table
- stone_pickaxe / stone_sword: requires wood + stone + nearby table + furnace

=== REWARD DESIGN REQUIREMENTS (be explicit and implement them) ===

Implement reward as a weighted sum of these components. Use deltas between current_step and prev_step.

1) BIG SPARSE BONUS FOR NEW ACHIEVEMENTS (main driver)
- When any achievement changes from 0 to >=1 this step, add a large positive bonus.
- Use higher bonuses for later/harder achievements, but do NOT make early achievements so valuable that the agent farms them forever.
- Do NOT reward "wake_up" (or give it ~0) since it is automatic.

Suggested achievement bonus weights (you may hardcode a dict):
- Early gather/sustain: collect_wood 3, collect_sapling 3, collect_drink 3, eat_plant 4, eat_cow 4
- Build/craft early: place_table 8, make_wood_pickaxe 10, make_wood_sword 6

- Late: collect_iron 16, make_iron_pickaxe 20, make_iron_sword 16
- Endgame: collect_diamond 30
- Combat: defeat_zombie 12, defeat_skeleton 14
(These are shaping suggestions; keep magnitudes comparable and stable.)

2) DENSE SHAPING FOR INVENTORY PROGRESS (smaller than achievements)
Reward ONLY positive, meaningful progress and cap it so it cannot be farmed indefinitely.
- Give small reward for increasing key materials/tools, but only up to a target amount.

- Reward tool acquisition (wood_pickaxe/stone_pickaxe/iron_pickaxe and swords) for first 1 unit only (since multiples are not needed).
- Use delta of clipped inventory: (min(cur, target) - min(prev, target)) so repeated collecting past target gives 0 reward.

3) STAGED / CURRICULUM BIAS (critical)
Add an extra small multiplier/bonus that focuses the agent on the NEXT missing milestone:
- If place_table not yet achieved: prioritize collecting wood (up to target) and successful table placement.
- Else if make_wood_pickaxe not yet achieved: prioritize wood (and attempting to craft) and crafting success.
- Else if collect_stone/coal not achieved: prioritize stone/coal gains.
- Else if place_furnace not achieved: prioritize stone and furnace placement.
- Else if make_stone_pickaxe not achieved: prioritize iron prerequisites and crafting success.
- Else if collect_iron not achieved: prioritize iron gains.
- Else: continue toward iron tools, diamond, combat.

Implement this as simple logic that increases weights for certain inventory deltas / achievement unlocks depending on what is still locked.

4) SURVIVAL SHAPING (must prevent dying while crafting)
- Strong penalty on death (terminated=True): e.g., -25.
- Small per-step alive bonus (e.g., +0.01) is OK.
- Add penalties when vitals are critically low to force eating/drinking/sleeping:
  * If food<=1: penalty
  * If drink<=1: penalty
  * If energy<=1: penalty

- Add a small positive reward for increasing food/drink/energy/health (e.g., when the agent eats/drinks/sleeps), but keep it smaller than achievement bonuses.

5) ANTI-WASTE / ACTIVITY
- Penalize noop (action==0) slightly.
- Penalize failed craft/place actions: if action is in {7..16} and BOTH achievements and inventory are unchanged vs prev_step, apply a small negative reward.
- Encourage movement a tiny amount (pos change) ONLY if nothing else changed, to reduce getting stuck (very small, e.g., +0.001).

6) SAFETY / STABILITY
- Handle prev_step=None by returning 0.0.
- Keep reward magnitudes bounded. After summing components, clip final reward to a reasonable range (e.g., [-30, +30]) to stabilize RL.
- Use only Python builtins.
